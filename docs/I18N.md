# ShareGate 国际化（i18n）

**所有版本的新增/修改 UI 与面向用户的文案都必须支持国际化。** 默认语言为 **English**；**简体中文** 为完整翻译。

## 语言文件

| 文件 | 用途 |
|------|------|
| `l10n/en.json` | 英文（源语言，key = 英文句子） |
| `l10n/zh_CN.json` | 简体中文 |
| `l10n/en.js`, `l10n/zh_CN.js` | 浏览器翻译（由 `npm run l10n` 从 JSON 生成；**必须**随应用部署） |

**同一 PR 中**：凡新增或修改用户可见字符串，**必须同时更新** `en.json` 与 `zh_CN.json`，然后运行 `npm run l10n` 重新生成 `.js`。

JSON 结构（Nextcloud 要求）：

```json
{
  "translations": {
    "English source string": "Translated string"
  },
  "pluralForm": "nplurals=2; plural=(n != 1);"
}
```

## 仓库文档（GitHub）

GitHub **不支持**运行时切换 README 语言；采用双文件约定：

| 文件 | 语言 |
|------|------|
| `README.md` | English（默认） |
| `README.zh-CN.md` | 简体中文 |

GitHub 会根据访问者语言自动展示对应 README（[Localized READMEs](https://docs.github.com/en/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/about-read-standards#localizing-readmes)）。修改功能说明时**请同步更新两个文件**。

应用内 UI 仍使用 `l10n/*.json`，与 README 无关。

## 前端（Vue / JS）

- 使用 `@nextcloud/l10n` 的 `translate('sharegate', key)`，或封装 `src/utils/l10n.js` 的 `t(key)`。
- **key 必须是英文**，且存在于 `l10n/en.json`。
- **不要**在 `t()` 里写中文 fallback；缺翻译时显示英文 key。

```javascript
// Good — shared helper
import { t } from '../utils/l10n.js'
{{ t('Paid shares') }}

// Bad — 英文 locale 下 en 翻译与 key 相同时会误用中文 fallback
{{ t('Paid shares', '付费分享') }}
```

新增 key 时先写入 `l10n/en.json`，再写入 `zh_CN.json`。

## 后端（PHP）

- 用户可见文案：`$this->l10n->t('English message')` 或注入 `IL10N`。
- **禁止**在 API `error` 字段、异常消息、JSON 响应中硬编码中文（或任何非英文源字符串）。
- 注释、日志可以是中文；**返回给浏览器的文本不行**。

## `appinfo/info.xml`（App Store 列表页）

[App Store 开发者文档](https://nextcloudappstore.readthedocs.io/en/latest/developer.html) 规定：

- **英文默认**：`<name>` / `<summary>` / `<description>` **不要**写 `lang` 属性（或写 `lang="en"`，但推荐省略）
- **简体中文**：另加一条 `lang="zh-hans"`（**禁止** `zh_CN`）
- **顺序**：先英文默认，再 `zh-hans` 翻译

```xml
<name>Paid sharing</name>
<name lang="zh-hans">付费分享</name>
<summary>Paid file sharing — …</summary>
<summary lang="zh-hans">付费文件分享 — …</summary>
<description><![CDATA[English markdown…]]></description>
<description lang="zh-hans"><![CDATA[中文…]]></description>
```

商店按访问者语言显示；无匹配时回退到**无 `lang` 的英文**。若只写 `lang="en"` + `lang="zh-hans"` 而无默认项，部分环境会错误显示中文。

与 `l10n/zh_CN.json`（应用内 UI）无关，勿混用语言码。

**注意**：`<navigations><navigation>` 内的 `<name>` 在 XSD 中**只能出现一次**，且**不支持** `lang` 属性（侧栏入口名由 NC 按应用 `l10n` 或英文 `name` 显示）。多语言仅写在应用顶层的 `<name>` / `<summary>` / `<description>`。

## 发布前检查

- [ ] 新增字符串已写入 `en.json` + `zh_CN.json`，并运行 `npm run l10n` 生成 `.js`
- [ ] Vue/JS 无中文 fallback
- [ ] PHP API 错误经 `IL10N`
- [ ] `info.xml`：英文用**无 `lang`** 的 `name`/`summary`/`description`，中文用 `lang="zh-hans"`
- [ ] 版本发布时同步更新 `CHANGELOG.md` 与 `CHANGELOG.zh-hans.md`（商店 `zh-hans` 更新日志）
- [ ] 个人语言设为 **English** 走一遍管理台与买家页
- [ ] 个人语言设为 **简体中文** 再走一遍
